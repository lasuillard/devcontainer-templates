# 👥 Development guide

This project is for personal use and not open for public contributions. Here, I describe how to develop and test Dev Container Templates locally for future reference.

## 🛠️ Tech stack

This project uses the following tech stack:

- [Bash](https://www.gnu.org/software/bash/)
- [Dev Container CLI](https://github.com/devcontainers/cli)

## 📂 Key directory structure

- `.devcontainer.example/`: Development environment configuration
- `.vscode.example/`: Project-specific VS Code configuration example
- `scripts/`: Helper scripts for development, testing and release
- `src/`: Source of each template
- `test/`: Tests for each template
- `flake.lock`, `flake.nix`: Flake configuration for development environment
- `Justfile`: Commands for development

## 🔧 Set up development environment

For development, you need to have the following tools installed:

### 🐋 System requirements

You would need to have [Docker](https://www.docker.com/) installed. This is not installable via Nix Flakes, so you need to install it manually on your host machine.

### ❄️ Tools managed via Nix Flakes

This repository uses [Nix Flakes](https://nix.dev/concepts/flakes.html) to manage tools. Following tools will be automatically installed (you need `nix` installed, of course):

- `devcontainer`
- [Just](https://just.systems/) (`just`)
- `pre-commit`
- `shellcheck`
- `shfmt`

We have configuration ([devcontainer.json](./.devcontainer.example/devcontainer.json)) for it, with Nix and Docker (DinD) installed. All you need to do is to run `nix develop` to start the development environment then run `just install` to install dependencies.

## ⌨️ Adding a new template

You can add new template in the following way:

- Copy and paste existing template in `src/` directory and make changes to it
- Add tests in `test/` directory
- Run `just test <template-id>` to test the template

New template will be automatically caught by automation workflow and you don't need to update any registry manually.

> [!NOTE]
> You don't need to make the documentation complete right away. The documentation will be automatically generated by release workflow, you can autogenerate draft by running `devcontainer templates generate-docs` at `src` directory though.

## ✅ Verifying changes

Before you push your code, you need to verify your code changes if it follows with the project's coding standard. You can run `just ci` to run all the necessary linters, formatters, and **SOME** tests. Only specific templates are tested because testing template could be a bit time-consuming job.

## 🚀 Releasing templates

Once code is pushed to the repository, templates can be released via [release.yaml](./.github/workflows/release.yaml) workflow. You should manually dispatch the workflow to trigger the release process.
