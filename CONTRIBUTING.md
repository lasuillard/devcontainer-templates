# 👥 Development guide

This project is for personal use and is not open to public contributions. This document describes how to develop and test Dev Container Templates locally for future reference.

## 🛠️ Tech stack

This project uses the following tech stack:

- [Bash](https://www.gnu.org/software/bash/)
- [Dev Container CLI](https://github.com/devcontainers/cli)

## 📂 Key directory structure

- `.devcontainer.example/`: Development environment configuration
- `.vscode.example/`: Project-specific VS Code configuration example
- `scripts/`: Helper scripts for development, testing, and release
- `src/`: Source of each template
- `test/`: Tests for each template
- `flake.nix`: Flake configuration for development environment
- `Justfile`: Commands for development

## 🔧 Set up development environment

For development, you need to have the following tools installed:

### 🐋 System requirements

You need to have [Docker](https://www.docker.com/) installed. This is not installable via Nix Flakes, so install it manually on your host machine.

### ❄️ Tools managed via Nix Flakes

This repository uses [Nix Flakes](https://nix.dev/concepts/flakes.html) to manage tools. The following tools will be automatically installed (you need `nix` installed):

- `pre-commit`
- `just`
- `devcontainer`
- `shellcheck`
- `shfmt`

Run `nix develop` to start the development environment, then run `just install` to install dependencies.

We also provide a dev container configuration ([devcontainer.json](./.devcontainer.example/devcontainer.json)) with Nix and Docker (DinD) installed.

## ⌨️ Adding a new template

You can add a new template as follows:

- Copy an existing template in the `src/` directory and modify it.
- Add tests to the `test/` directory.
- Run `just test <template-id>` to test the template.

New templates will be automatically detected by the automation workflow; you don't need to update any registry manually.

> [!NOTE]
> You don't need to complete the documentation right away. The documentation will be automatically generated by the release workflow; if you want, you can autogenerate a draft by running `devcontainer templates generate-docs` in the `src` directory.

## ✅ Verifying changes

Before you push your code, verify that your changes follow the project's coding standards. You can run `just ci` to execute the necessary linters, formatters, and some tests.

Because testing templates can be time-consuming, only specific templates are tested locally. The CI workflow run tests for all templates.

## 🚀 Releasing templates

Once code is pushed to the main branch, templates will be released automatically via the [release.yaml](./.github/workflows/release.yaml) workflow if the `devcontainer-template.json` file is updated.
